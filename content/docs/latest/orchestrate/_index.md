# Flatcar Orchestration & Container Runtimes

Flatcar Container Linux supports all of the popular methods for running
containers, and you can choose to interact with the containers at a
low-level, or use a higher level orchestration framework. Listed below are
some guides to help you choose and make use of the different runtimes.

- [Getting started with Docker](./containers/getting-started-with-docker.md)
- [Customizing Docker](./containers/customizing-docker.md) 
- [Using systemd to manage Docker containers](../os-config/systemd/getting-started.md)
- [Authenticating to Container registries](./containers/registry-authentication)
- [Getting started with Kubernetes](./kubernetes/getting-started-with-kubernetes.md)
- [High availability Kubernetes](./kubernetes/getting-started-with-kubernetes.md)

## Containers

Docker and containerd ship as built-in sysext images enabled by default.
